# CompuTec Quality Testing Mobile (QC360) Plugin

Below are the release notes for CompuTec Quality Testing Mobile (QC360) Plugin, highlighting new features, improvements, and bug fixes in this release.

## CompuTec QC360 3.2606.1

**Release Date: 18 June 2026**

| Issue Type | Component | Release Note |
| --- | --- | --- |
| New | QC360 | Added support for saving and reusing personal filter sets in **QC360**, making it easier to quickly access frequently used **Quality Control** views. |
| Improved | QC360 | Improved search and filtering behavior for more consistent and reliable results. |
| Improved | QC360 | **QC Test Pool** status fields are now strongly typed using dedicated status enums. This improves data consistency and simplifies development for integrations and custom extensions. |
| Improved | Localization | Added Polish language translations. |

:::warning[important]

**CompuTec QC360 3.2606.1** requires:

    - **CompuTec ProcessForce 3.2606.3** or higher
    - **CompuTec Core 3.0.1.577** or higher

:::

:::info[for developers]
If you maintain custom integrations or external plugins that interact with **CompuTec QC360 Plugin**, you must recompile and update your code.

| Previous Value | New Value |
| --- | --- |
| `udo.U_Status = "CR"` | `udo.U_Status = QCPoolDocumentStatus.Created` |
| `udo.U_TestStatus = "FL"` | `udo.U_TestStatus = QCTestStatus.Failed` |

For scenarios where status values are received as raw strings, use:
        - ``QCPoolDocumentStatusConverter``
        - ``QCTestStatusConverter``

:::
